先把宿主机目录、端口和依赖服务准备好，再启动 Compose。

来源：`doc/docker-compose-deploy.md` 目标域名：`doc.infoq.cc` 最近更新：2026-04-27

这篇文档应该先回答什么

正文页不应该只堆命令，而要先告诉运维或开发者“为什么要这样部署、前置条件是什么、失败时先看哪里”。

1. 准备宿主机目录

部署前先建立统一宿主机根目录，保持 MySQL、Redis、MinIO、Nginx、server config 和日志路径可预期，避免脚本运行时静默创建到错误位置。

`INFOQ_DEPLOY_ROOT` 明确存在时优先使用；否则脚本会回退到 `/infoq` 或用户目录。正文应明确写出这一行为，避免部署者误判真实落盘路径。

2. 运行部署脚本

正文页中的命令块必须突出显示，并与段落说明分开，方便维护者直接复制，同时保留解释文字。

export INFOQ_DEPLOY_ROOT=/tmp/infoq-deploy bash script/bin/infoq.sh prepare bash script/bin/infoq.sh deploy bash script/bin/deploy-frontend.sh prepare bash script/bin/deploy-frontend.sh deploy

3. 失败时先看哪里

  • 后端容器日志和端口是否正常暴露
  • MySQL / Redis 是否真正就绪，而不是容器已启动但服务未可用
  • 初始化 SQL 是否导入完成
  • Nginx 是否把 `/vue/`、`/react/`、`/prod-api/` 路由到正确位置
