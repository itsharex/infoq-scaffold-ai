# 检查清单与反模式

## 格式检查清单

- 第一行是否为 `# AGENTS.md`。
- 顶部是否包含 retrieval-led `|IMPORTANT:`。
- 标题后非空行是否全部 `|` 开头。
- 是否移除了 `##`/`###` 与代码围栏。

## 内容检查清单

- 是否覆盖项目根、模块/工作区、配置文件、docs root、workspace `AGENTS.md`、skill root、构建/测试命令。
- 路径是否真实存在且拼写准确。
- 是否包含提交规范与 PR 检查项。
- 长 SOP 是否已下沉到 `doc/` 或 `references/`。

## 精简度检查清单

- 是否把解释性文字压缩为索引事实和检索入口。
- 是否避免一行承载多个不相关主题。
- 行数是否在可维护范围（建议 15-35 行）。

## 反模式

- 在根 `AGENTS.md` 中回流长篇章节散文，导致被动上下文膨胀。
- 把完整 SOP 塞进 `AGENTS.md`，而不是下沉到可检索文件再做索引。
- 写入不存在的路径、不可检索的 docs root 或“猜测式”命令。
- 缺少 retrieval-led `|IMPORTANT:` 导致 agent 优先回退到预训练记忆。
- 把动作型 workflow 写进根 `AGENTS.md`，而不是沉淀为独立 skill。
